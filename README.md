# JobHunt Agent

Мультиагентная система для анализа вакансий и генерации персонализированных
сопроводительных писем. Помогает соискателю быстро отбирать релевантные
позиции, делать gap-анализ навыков и автоматически готовить отклики.

> **Статус:** MVP в активной разработке. Базовый пайплайн (парсинг → векторизация → анализ → генерация cover letter) работает; ведётся доработка eval-pipeline и Telegram-интерфейса.

---

## Зачем

Поиск работы — это сотни вакансий, ручное сравнение с резюме и долгое
написание сопроводительных. Шаблонные cover letters не работают —
рекрутёры их фильтруют. Цель проекта — собрать систему, которая:

- ежедневно подтягивает свежие вакансии под заданные запросы;
- сравнивает их с резюме пользователя по векторной близости и LLM-судье;
- выявляет пробелы в навыках и приоритизирует, что подтянуть;
- генерирует персонализированное сопроводительное письмо под каждую вакансию;
- доступна как Telegram-бот для быстрой работы с телефона.

```
│  hh.ru API     │───▶│  Vacancy        │───▶│  Vector Store    │
│  (parser)      │    │  Preprocessor   │    │  (Qdrant)        │
└────────────────┘    └─────────────────┘    └──────────────────┘
                                                       │
                                                       ▼
┌────────────────────────────────────────────────────────────────┐
│                    LangGraph Agent Orchestration               │
│                                                                │
│   Researcher → Gap-Analyzer → Writer → Critic → Finalizer      │
└────────────────────────────────────────────────────────────────┘
                                                       │
                       ┌───────────────┬───────────────┴──┐
                       ▼               ▼                  ▼
                 ┌──────────┐   ┌────────────┐    ┌─────────────┐
                 │ Eval     │   │ FastAPI    │    │ Telegram    │
                 │ Pipeline │   │ Service    │    │ Bot         │
                 │ (LLM-as- │   │            │    │ (aiogram)   │
                 │ judge)   │   │            │    │             │
                 └──────────┘   └────────────┘    └─────────────┘
```

### Агенты

- **Researcher** — извлекает ключевые требования, стек и грейд из текста вакансии.
- **Gap-Analyzer** — сопоставляет требования с резюме пользователя, выдаёт список совпадений и пробелов.
- **Writer** — пишет первый draft сопроводительного письма с упором на match-навыки.
- **Critic** — проверяет письмо по чеклисту (релевантность, длина, тон, отсутствие штампов) и предлагает правки.
- **Finalizer** — собирает финальную версию, готовит к отправке.

### Eval Pipeline

Метрики качества cover letter:
- **Relevance** (LLM-as-judge): насколько письмо отражает требования вакансии.
- **Personalization**: степень упоминания конкретики (название компании, продукт, стек).
- **Length & Structure**: попадание в целевой диапазон, наличие хука и CTA.
- **Hallucination check**: проверка, что не выдумано опыта, которого нет в резюме.

## Стек

**Core:**
- Python 3.11
- LangGraph + LangChain (оркестрация агентов)
- Qdrant (векторное хранилище)
- OpenAI API / Anthropic Claude (для разработки и сравнения качества)
- YandexGPT 5 Pro (для прода в РФ-инфраструктуре)
- Pydantic (схемы данных)

**Инфра:**
- FastAPI (REST API)
- aiogram (Telegram-бот)
- Docker, docker-compose
- pytest + LLM-eval

**Источники данных:**
- hh.ru API (вакансии)
- Резюме пользователя в md/json формате

## Установка и запуск

